# Сервис выгрузки архива погоды RP5

Это готовый каркас веб-сервиса для ручного запуска выгрузки погодных архивов с редактируемым списком городов.

## Что уже есть
- Веб-интерфейс на русском.
- Добавление и удаление городов.
- Ручной запуск выгрузки кнопкой.
- Excel на выходе: `README`, `ALL_DATA`, отдельные листы по городам.
- Хранение данных в SQLite.
- Готовность к деплою на Render через `render.yaml`.

## Важное ограничение MVP
Сейчас внутри проекта стоит **заглушка загрузчика** RP5. Это значит:
- интерфейс, база, экспорт в Excel и деплой уже готовы;
- модуль получения данных из RP5 нужно подключить отдельно после проверки фактического механизма скачивания архива.

Это сделано специально, чтобы не закладывать неустойчивый парсер без теста реальных сетевых запросов и ограничений сайта.

## Структура
- `app/main.py` — веб-приложение FastAPI.
- `app/parser.py` — модуль загрузки данных из RP5 (сейчас заглушка).
- `app/excel_export.py` — сборка Excel.
- `app/templates/index.html` — интерфейс.
- `app/static/styles.css` — стили.
- `render.yaml` — конфиг Render.

## Локальный запуск
```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

## Переменные окружения
- `DATA_DIR` — путь для БД и файлов, локально можно не задавать.
- `APP_PASSWORD` — зарезервировано на будущее для авторизации.

## Как задеплоить на Render
1. Создай новый репозиторий на GitHub.
2. Залей в него все файлы проекта.
3. В Render нажми **New +** → **Blueprint**.
4. Подключи GitHub-репозиторий.
5. Render прочитает `render.yaml` и предложит создать web service.
6. Подтверди деплой.
7. После создания сервиса проверь, что подключен disk `weather-data` на `/var/data`.
8. После первого запуска открой URL сервиса.

## Что делать дальше
Чтобы сервис стал рабочим именно как загрузчик архива RP5, нужно доработать `app/parser.py` одним из способов:
- через прямой HTTP-сценарий, если удастся стабильно повторить запрос скачивания архива;
- через Selenium/Playwright, если RP5 требует браузерного сценария.
