# Сервис выгрузки архива погоды RP5

Версия без локальных образцов: сервис работает только через онлайн-получение данных с RP5.

## Что изменено
- Убраны все fallback на приложенные XLS/XLSX-образцы.
- `parser.py` работает только через `weather-rp5`.
- Данные запрашиваются по URL станции RP5 и диапазону дат.
- Результат нормализуется в единую схему колонок для корректного общего листа `ALL_DATA`.

## Как это работает
1. Пользователь добавляет город и URL архива RP5.
2. Сервис получает `station_id` через `get_station_id()`.
3. Затем получает архив через `get_weather_data()`.
4. После этого приводит результат к канонической таблице колонок.
5. Excel собирается без сдвига полей между городами.

## Локальный запуск
```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

## Важно
Если RP5 или пакет `weather-rp5` не смогут вернуть данные по конкретной станции, сервис покажет ошибку по этому городу в логе запуска. Локальных резервных файлов в этой версии больше нет.
